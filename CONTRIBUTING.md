# Contributing

Thanks for helping improve StoryVista.

## How to report bugs

Open a bug report and include:

- What you tried to do
- What input type you used
- What happened
- What you expected
- Screenshots or sanitized examples if helpful

Do not include private manuscripts or sensitive local paths.

## Suggest templates

Templates are welcome for:

- Novels
- Screenplays
- Actor preparation
- RPG settings
- Obsidian archives
- Static story atlases

## Improve documentation

Documentation pull requests should be clear, practical, and example-driven.

## Pull request process

1. Open an issue or describe the change clearly in the PR.
2. Keep changes focused.
3. Update docs when behavior or usage changes.
4. Run skill validation if the `skill/` folder changes.

## Style

- Avoid hype.
- Prefer real examples.
- Keep privacy and copyright risks visible.
- Use inclusive, globally understandable language.
