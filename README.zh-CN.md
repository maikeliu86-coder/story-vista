# StoryVista｜文景

## 让文字世界看得见

![StoryVista 横幅：手稿、剧本和小说页转化为人物图谱、时间线、场景地图和世界图谱](assets/github-hero.png)

**StoryVista｜文景** 是一个 Codex skill，用来把小说、剧本、长篇文本、角色设定、世界观资料、跑团设定和创作笔记，转化为可以浏览、理解和复盘的可视化故事图谱。

它帮助读者、作者、编剧、演员、导演、制片、学生、老师、研究者和世界观创作者，把复杂文字整理成人物关系、场景地点、时间线、阵营势力、物件线索、概念设定、角色弧光和 3D 世界结构。

英文说明：[README.md](README.md)

## 为什么需要 StoryVista

很多长篇故事的问题不是“读不懂一句话”，而是“装不下整个世界”。

人物有别名，关系会反转，地点不断跳转，物件和伏笔分散在不同章节，剧本里一句台词背后可能藏着完整的人物关系和世界结构。

StoryVista 的目标，是把这些隐藏在文字里的结构做成可以看的东西。

## StoryVista 能生成什么

- 人物关系图
- 人物索引和角色小传
- 场景地图
- 地点路线图
- 时间线
- 阵营势力图
- 物件、线索、符号、母题索引
- 世界观设定图谱
- 演员案头工作图谱
- Obsidian 本地故事档案
- 可公开分享的静态页面
- 当空间关系重要时，生成 3D 世界图谱

## 修正版 Skill 新增重点

新版 StoryVista｜文景 默认面向“可交互、可探索、可在多端打开”的故事档案，而不是普通文字总结。

- **响应式优先**：所有 HTML / 网页产物都要适配电脑、平板和手机，保证文字可读、点击区域足够大、移动端滚动不被图谱无故锁死。
- **先读文本再分类**：人物、地点、飞船、技术、能力、组织、物品和线索必须分开建模。飞船、地点、组织不能误放进人物关系树。
- **继承优秀模板**：如果用户提供已有优秀页面，应优先复刻它的排版气质和交互逻辑，再替换为新文本的数据。
- **人物图谱更清晰**：人物缩略图使用独立人物卡片；人物关系树优先按阵营、功能或故事线分区；点击人物后高亮相关人物和关系。
- **真正的 3D 空间图**：星球、飞船、城市、基地、空间站、路线和世界节点应做成有体积、有运动、有点击区域的 3D 微缩模型，而不是贴在 3D 空间里的二维照片卡片。
- **跨设备 3D 手势**：3D 地图旋转应围绕网格/地图中心，而不是某个节点；缩放应以指针位置为中心；手机端纵向划过地图时不应卡住页面滚动，只在明确横向拖动或双指操作时接管地图。

## 演员如何使用 StoryVista

对于演员，StoryVista｜文景 可以把剧本转化为案头工作图谱：

- 梳理角色关系
- 找人物前史
- 找欲望、阻碍、行动
- 拆分每一场戏的任务
- 追踪人物弧光
- 找对手戏关系
- 建立人物地图
- 辅助角色小传和案头工作

示例：

```text
请使用 $story-vista 阅读这份剧本，帮我生成一份演员案头工作图谱：人物关系、每场戏的任务、角色欲望、阻碍、行动、情绪变化和人物弧光。
```

## 编剧、导演和制片如何使用 StoryVista

- 追踪物件和伏笔
- 给团队生成可讨论的故事地图

示例：

```text

```

## 快速开始

把 `skill` 文件夹复制到 Codex 的 skills 目录：

```bash
mkdir -p "$HOME/.codex/skills"
cp -R skill "$HOME/.codex/skills/story-vista"
```

之后在新的 Codex 会话中这样调用：

```text
请使用 $story-vista，把这部小说、剧本或长篇文字做成人物、关系、场景、地点、时间线和世界观概念的交互式视觉档案。
```

更多说明见：[docs/quickstart.md](docs/quickstart.md)。

## 工作方式

1. 读取源文本。
2. 识别叙事实体。
3. 区分人物、地点、物件、阵营、时间线、母题和概念。
4. 生成结构化故事图谱。
5. 按需生成或绑定视觉素材。
6. 验证导航、图片匹配、实体分类和公开分享链接。

## 设计与验收原则

- 以文本证据为准，不用视觉感觉代替实体分类。
- 人物、地点、飞船、技术、组织、物件必须分开处理。
- 人物头像、人物名字、关系节点、3D 模型本体和标签都应该能点击。
- 人物卡片不能拉伸变形，移动端必须显示完整中英文名且不遮挡人脸。
- 人物关系树要避免杂乱网状图，优先按阵营或功能分区。
- 3D 空间图禁止使用二维贴片、圆角照片卡片、相册墙或伪 3D。
- 3D 空间图旋转要围绕网格/地图中心，移动端纵向滑动页面不能被地图误拦截。
- 发布前应检查桌面、平板、手机三种视口，确认布局、点击和缩放交互可用。

## 局限

- 输出质量取决于源文本是否清楚完整。
- 长篇作品可能需要分块处理和多轮校正。
- 配图生成和人物/地点匹配需要用户复核。
- 别名复杂、非线性叙事、不可靠叙述会增加识别难度。
- StoryVista 不能替代文学判断、编剧判断、表演选择或人类创作决策。
