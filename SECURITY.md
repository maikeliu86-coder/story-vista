# Security and Privacy

StoryVista often works with unpublished manuscripts, scripts, notes, and local archives. Treat source material carefully.

## Do not post sensitive material publicly

Do not include these in public issues, pull requests, screenshots, or examples:

- Private manuscripts
- Unpublished scripts
- NDA-protected material
- Copyrighted text you do not have permission to share
- API keys or tokens
- Sensitive personal data
- Local file paths that reveal private information

## Safe examples

Use public-domain texts, short synthetic examples, or heavily sanitized excerpts.

## Reporting security issues

If you find a security or privacy issue, report it privately to the repository owner rather than opening a public issue.
