# FAQ

## Is StoryVista only for novels?

No. It works for novels, scripts, screenplays, RPG settings, long-form prose, story bibles, and dense creative notes.

## Can it handle screenplays?

Yes. It can map characters, scene objectives, locations, conflicts, emotional beats, and production-facing structure.

## Can actors use it for role preparation?

Yes. StoryVista can help build a role-preparation atlas: relationships, objectives, obstacles, actions, emotional shifts, opponent relationships, and character arcs.

## Does it generate images?

It can use Image2 / GPT-Image workflows when requested. Generated images should be reviewed and corrected before public use.

## Does it work with Obsidian?

Yes. StoryVista supports Obsidian-ready local HTML archives and source-record notes.

## Can I publish the output?

Yes, if you have the rights to the source material and remove sensitive or copyrighted content that should not be public.

## Does it work with Chinese texts?

Yes. The documentation includes Chinese examples, and the workflow can be used with Chinese novels, scripts, and notes.

## How do I contribute?

Open an issue or pull request with bug reports, templates, documentation improvements, example workflows, or use cases.

## Does StoryVista replace literary analysis?

No. It helps structure and visualize material, but interpretation remains a human responsibility.

## Can it handle very long books?

Yes, but long works may need chunking, review passes, and correction of aliases or entity categories.
