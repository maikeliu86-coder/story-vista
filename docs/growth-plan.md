# 30-Day Ethical Growth Plan for StoryVista

## Rules

- Do not buy stars.
- Do not request fake stars.
- Do not spam communities.
- Do not exaggerate.
- Show real demos.
- Ask for feedback before asking for stars.

## Week 1

- Finish public-readiness cleanup.
- Polish README and topics.
- Prepare first release.
- Add demo screenshots.
- Pin StoryVista on GitHub profile.
- Add StoryVista to profile README.

## Week 2

- Publish one example atlas.
- Create a short demo video.
- Share carefully on X/Twitter.
- Share on LinkedIn.
- Share in writing, screenwriting, AI workflow, and worldbuilding communities only where relevant.

## Week 3

- Collect feedback.
- Open roadmap issues.
- Add templates from real use cases.
- Improve Chinese docs.
- Add actor-preparation example.

## Week 4

- Release v0.1.1.
- Share a case study.
- Publish a post: `How I built StoryVista`.
- Invite contributors.
