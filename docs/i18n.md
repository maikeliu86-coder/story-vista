# Internationalization

StoryVista separates source language from interface language. English source can use Chinese UI; Chinese source can use English UI. Japanese or Russian source can currently use English or Chinese UI while script-aware extraction remains experimental.

`language-profile.json` records input language, script, writing direction, name system, UI language, translation mode, and locale status. RTL structure is reserved for Arabic and Hebrew text even though full extraction is not yet implemented.

Proper nouns retain `canonical_name`. Optional `localized_names`, `localized_aliases`, and `memory_label` fields improve recall without replacing the original spelling. Western title/surname rules and Chinese short-name rules are conservative; collisions remain ambiguous.

Supported UI locales: `en`, `zh-CN`. Experimental locale structures: `zh-TW`, `ja`, `ko`, `fr`, `es`, `de`, `ru`. Missing keys fall back to English. Experimental does not mean fully translated or linguistically validated.
