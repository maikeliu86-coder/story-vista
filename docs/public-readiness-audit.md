# Public-Readiness Audit

This audit records the public-facing cleanup for StoryVista's first public beta
documentation pass.

## What was already public-ready

- The repository already had a clear product name: `StoryVista｜文景`.
- The core tagline was strong: `Make Text Worlds Visible / 让文字世界看得见`.
- The repository already included bilingual README files.
- The repository already had visual assets for the hero, pain points, and workflow.
- The `skill/` folder was already structured as an installable Codex skill.

## What was confusing

- Some documentation framed the repository as a private or internal preview even
  though the repository is already public.
- The README began closer to a technical project note than a product landing
  page for readers, writers, actors, screenwriters, and worldbuilders.
- GitHub discovery materials were incomplete: topics, description alternatives,
  release notes, contribution guidance, issue templates, and growth plan were
  not fully represented.

## What was missing

- Public beta positioning.
- Practical quick-start docs.
- Example prompt library.
- FAQ, roadmap, and contribution documents.
- GitHub issue and pull request templates.
- Recommended repository description and topic set.
- Ethical growth plan and profile promotion guidance.
- v0.1.0 release notes draft.

## What changed

- Rewrote the English and Chinese README files for public onboarding.
- Removed draft/private/internal-review language from public docs.
- Added public beta language, limitations, privacy guidance, and contribution
  links.
- Added documentation under `docs/` for quick start, examples, prompts, FAQ,
  roadmap, keywords, visual assets, growth, profile promotion, and release notes.
- Added GitHub community files for issues, pull requests, contributing,
  changelog, and security.

## Remaining manual settings

- Choose and add a license before broad distribution.
- Pin the repository on the GitHub profile.
- Add screenshots or demo GIFs when real example atlases are available.
- Create a GitHub release only after the public beta branch is merged or accepted.
